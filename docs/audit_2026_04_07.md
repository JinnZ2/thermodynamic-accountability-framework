## CRITICAL ŌĆö Won't Run

### 1. Smart Quotes in Python Files (8 files)

Unicode curly quotes (`U+201C` `"`, `U+201D` `"`) appear throughout 8 files, causing `SyntaxError: invalid character` on import.

**Affected files:**
- `core/atbs/functional_detector.py`
- `core/atbs/test_v2.py`
- `simulations/full_coupled_system.py`
- `simulations/federation.py`
- `simulations/emergent_federation.py`
- `simulations/node_v3_ipi.py`
- `game_theory/proof-pipeline.py`
- `game_theory/proof-pipelineV2.py`

**Fix:** Replace all `"` `"` with ASCII `"`, and `'` `'` with `'`.

### 2. Markdown Code Fences in Python (8 files)

Triple backtick fences (` ``` `) appear inside Python source, from copy-paste out of markdown. Code inside these blocks is unreachable ŌĆö Python treats ` ``` ` as a syntax error or a string literal depending on context.

**Same 8 files as above.**

**Fix:** Remove all ` ``` ` lines.

### 3. Mangled Dunder Methods (5 files)

`__init__`, `__dict__` appear as `**init**`, `**dict**` ŌĆö markdown bold formatting leaked into Python source.

**Affected files and examples:**
- `simulations/full_coupled_system.py`: `n.**dict**` instead of `n.__dict__`
- `simulations/federation.py`: `FedNode(**n.**dict**)` instead of `FedNode(**n.__dict__)`
- `simulations/node_v3_ipi.py`: `s.**dict**` instead of `s.__dict__`
- `game_theory/proof-pipeline.py`: `def **init**(self):` instead of `def __init__(self):`
- `game_theory/proof-pipelineV2.py`: same pattern

**Fix:** Replace `**init**` ŌåÆ `__init__`, `**dict**` ŌåÆ `__dict__`, etc.

## HIGH ŌĆö Logic / Design Bugs

### 4. Dataclass Default Evaluated at Class Time (`simulations/seed_sim.py`)

```python
@dataclass
class Community:
    K_kinesthetic: float = np.random.uniform(0.5, 0.9)  # line 13
    K_temporal:    float = np.random.uniform(0.5, 0.9)  # line 14
    ...
```

`np.random.uniform()` is called **once** when the class is defined. Every `Community()` instance that doesn't override these fields gets the **same** value. All communities start identical.

**Fix:** Use `field(default_factory=lambda: np.random.uniform(0.5, 0.9))` or initialize in `__post_init__`.

### 5. Duplicate `HumanSystemModel` Class

Defined independently in both:
- `core/fatigue_model.py` (line 142) ŌĆö delegates to `FatigueModel`
- `core/human_system_collapse_model.py` (line 4) ŌĆö standalone implementation

Different method signatures (`compute_collapse_risk` vs `assess_system`), different return types (tuple vs dict). Which one should be used? The CLAUDE.md audit notes from 2026-03-24 say `HumanSystemModel` now delegates to `FatigueModel` (DRY), but the standalone version in `human_system_collapse_model.py` still exists.

### 6. Division by Zero Risk

- `simulations/full_coupled_system.py` ~line 119: `(val-floor)/(1-floor)` ŌĆö crashes if `floor == 1.0`
- `simulations/federation.py` ~line 156: `3.0/n_nodes` ŌĆö crashes if `n_nodes == 0`

### 7. Missing Method Call Parentheses (`core/atbs/functional_detector.py`)

```python
datetime.now() - max(breaches, key=lambda i: i.timestamp).timestamp
```

Should be `.timestamp()` (with parens) ŌĆö without them, this subtracts a method object from a datetime, raising `TypeError`.

### 9. Stub Modules

`core/heat_leak_case.py` (32 lines) and `core/data_logger.py` (23 lines) are minimal stubs ŌĆö not fully realized implementations. They work but cover only basic scenarios.
