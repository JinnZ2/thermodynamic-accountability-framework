# Audit of newly added Mathematic-economics files

This note captures the initial audit of the files added on 2026-04-09 in `Mathematic-economics` and assesses whether they are ready to be transferred into `thermodynamic-accountability-framework`.

| File | Type | Audit status | Key findings | Recommended disposition |
|---|---|---:|---|---|
| `dynamic_cpi_indicator.py` | Python module | Partial / viable with dependency caveat | The file is structurally complete, uses `numpy` and `scipy`, contains a runnable backtest, and includes a simple API stub. However, it imports `softmax` without using it, imports `warnings` without using it, and mixes simulation logic with API concerns in one file. | Transfer only after placing it in a simulations-oriented economics area and separating documentation from executable code if possible. |
| `dynamic_inflation_weight.py` | Python script | Incomplete / not runnable | The file references multiple undefined names including `initial_weights`, `T`, `true_weights`, `price_changes`, `spend_changes`, `category`, `rent_idx`, `ce_release_time`, `ce_weights_previous`, `historical_volatility_of_ce`, and more. It appears to be pseudocode or an unfinished prototype rather than a functioning module. | Do not promote as production code. Preserve as an archival draft or convert into documentation/examples rather than executable code. |
| `iteration_module.py` | Python module fragment | Incomplete / not runnable | The file defines methods inside `DynamicCPI_R` but omits critical imports and state initialization. It uses `np` without importing `numpy`, references attributes such as `self.n`, `self.w`, `self.P`, `self.sigma_w`, `self.R_base`, and helper methods that are not established in the file, and contains placeholder comments indicating missing prior code. | Treat as an extension fragment for `dynamic_cpi_indicator.py`, not as a standalone module. Preserve as an experimental add-on or merge into a documented appendix folder. |
| `api.json` | JSON example | Invalid JSON | The file lacks the opening `{` brace and therefore is syntactically invalid. Its content is usable as a payload example once corrected. | Fix syntax and store as an example payload near the related CPI documentation or API examples. |

The target repository conventions indicate that `core/` should remain standard-library only, while scientific dependencies belong under simulation-oriented areas. That means the dynamic CPI code should not be dropped into `core/`. The economics documentation already lives under `docs/economics/`, so unfinished prototypes and API examples fit better under an economics subfolder than at repository root.
